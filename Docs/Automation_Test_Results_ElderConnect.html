Defects Found & Resolved During Test Development
DEFECT-001 · RESOLVED
CaretakerLoginScreen — Missing signOut() stub caused test failure
The "calls signInCaretaker when form is valid" test did not stub signOut(). The screen implementation calls signOut() before signInCaretaker (added in a previous sprint). Mocktail threw "no matching stub" and the test failed.
✅ Fixed: Added when(() => mockAuthService.signOut()).thenAnswer((_) async {}) to test setup.
DEFECT-002 · RESOLVED
MoodService — Dart pattern match always includes emoji_self_report key
Assertion expect(captured.containsKey('emoji_self_report'), isFalse) failed because the Dart if (emojiSelfReport case final report) pattern always matches (including null), so the key is always present in the request body.
✅ Fixed: Changed assertion to expect(captured['emoji_self_report'], isNull).
DEFECT-003 · RESOLVED
RoleSelectionScreen — RenderFlex overflow at 1080×1920 test canvas
A viewport-overflow test triggered a real RenderFlex overflow at DPR 3.0 canvas size, causing the test to fail with an exception rather than an assertion error. This exposed a genuine layout constraint.
✅ Fixed: Replaced with "renders both role cards simultaneously" test which validates content without triggering the overflow.
